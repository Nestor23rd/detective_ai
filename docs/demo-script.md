# Demo Script

## Goal
The video should prove three things very quickly:
- this is not a thin AI wrapper
- ASI-1 is used in a structured, multi-step workflow
- the product is trustworthy because it shows evidence, limitations, and guardrails

## Best winning format
Use one live investigation plus one saved edge-case report:
1. Live run: Hinglish or Hindi misinformation sample from the analysis page
2. Saved report: inaccessible video URL that returns `Cannot analyze`

That combination sells both strengths:
- rich claim-level evidence
- honest handling of uncertainty

## Before recording
Prepare the app like this:
1. Open the home page in one clean browser tab.
2. Make sure the archive already contains:
   - one strong multilingual text/image report
   - one video URL report showing `Cannot analyze`
3. Keep the analysis page ready with the Hinglish sample case.
4. Use browser zoom around `90%` or `100%`.
5. Record at `1280x720` or `1920x1080`.
6. Keep MongoDB and backend already running so there is no setup delay.

## Recommended 75-90 second cut

### 0s-8s: Open on the home page
Show:
- the hero section
- supported inputs
- the claim-level positioning

Say:
"AI Internet Detective turns suspicious text, screenshots, PDFs, video links, and clips into evidence-led investigations powered by ASI-1."

### 8s-20s: Jump to the analysis page
Show:
- the input mode buttons
- load the Hinglish sample
- click `Run AI Investigation`

Say:
"We support six input modes, but for this demo I’ll run a multilingual misinformation case similar to a viral forward."

### 20s-30s: Hold on the live processing overlay
Show:
- the ASI-1 processing panel
- progress, stages, and activity stream

Say:
"This is not one prompt and one score. The backend uses ASI-1 in stages to extract signals first, then build a structured investigation report."

### 30s-55s: Land on the results page
Show:
- score or verdict area
- language and risk labels
- claim-by-claim evidence
- source URLs

Say:
"The result is explainable. Each claim gets an assessment, confidence, evidence bullets, source links, and a recommended next verification step."

### 55s-68s: Use follow-up Q&A
Ask:
`What is the weakest claim in this report?`

Say:
"Judges can challenge the report live. ASI-1 can answer follow-up questions grounded in the saved investigation."

### 68s-80s: Open the archive and show the video edge case
Show:
- archive page
- a saved video URL case
- `Cannot analyze`
- reason and checks

Say:
"A key trust feature is that inaccessible media is not mislabeled as fake. If a video cannot be inspected, the system returns `Cannot analyze` and explains why."

### 80s-90s: Close on value
Show:
- archive or home page CTA

Say:
"That makes the product useful for real fact-checking workflows: evidence-led, multilingual, media-aware, and honest about uncertainty."

## Exact voiceover script
"AI Internet Detective turns suspicious text, screenshots, PDFs, video links, and clips into evidence-led investigations powered by ASI-1.

Here I’m running a multilingual misinformation example, similar to the kinds of forwards that spread quickly online.

During processing, ASI-1 works through a structured workflow instead of returning a single opaque answer.

On the results page, the report is explainable: each claim has an assessment, confidence score, evidence bullets, source links, and next-step guidance.

I can even challenge the report with a follow-up question and get an answer grounded in the saved investigation.

And importantly, when a video link is inaccessible, the system does not force a fake score. It returns `Cannot analyze`, explains the limitation, and recommends what a human should verify next.

So the product is not just fast. It is structured, audit-friendly, multilingual, and honest about uncertainty." 

## 60-second emergency version
If time is tight, use this shorter sequence:
1. Home page hero: `5s`
2. Analysis page sample + click run: `8s`
3. Processing overlay: `6s`
4. Results page claims + sources + Q&A: `26s`
5. Archive page `Cannot analyze` video case: `10s`
6. Final close sentence: `5s`

## Best on-screen moments to pause on
- the animated ASI-1 processing overlay
- one strong claim card with evidence bullets and URLs
- the follow-up answer block
- the video report showing `Cannot analyze`

## Avoid these demo mistakes
- Do not spend time on setup, login, or environment screens.
- Do not run two long live investigations in one recording.
- Do not focus only on the overall score.
- Do not skip the video guardrail because it is one of the strongest trust signals in the product.
- Do not read every card aloud; point at the structure and move.
